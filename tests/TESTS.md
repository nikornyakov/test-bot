# 🧪 Тестовая документация

## Обзор

Этот документ описывает систему тестирования проекта "Lets Play Basketball Bot". Проект включает комплексные unit-тесты, интеграционные тесты и тесты конфигурации для обеспечения надежности и качества кода.

## Структура тестов

```
tests/
├── __init__.py                    # Инициализация пакета тестов
├── test_bot_config.py            # Тесты конфигурации
├── test_bot_base.py              # Тесты базового класса бота
└── test_simple_bot.py            # Интеграционные тесты основных функций
```

## Настройка среды тестирования

### Зависимости

```bash
pip install -r requirements.txt
```

Требуемые пакеты для тестирования:
- `pytest==7.4.3` - Основной фреймворк тестирования
- `pytest-asyncio==0.21.1` - Поддержка асинхронных тестов

### Конфигурация pytest

Файл `pytest.ini` содержит настройки:
```ini
[tool:pytest]
testpaths = tests
python_files = test_*.py
python_classes = Test*
python_functions = test_*
addopts = -v --tb=short
asyncio_mode = auto
```

## Запуск тестов

### Все тесты
```bash
pytest
```

### Конкретный файл
```bash
pytest tests/test_bot_config.py
```

### Конкретный тест
```bash
pytest tests/test_bot_config.py::TestConfig::test_load_config_success -v
```

### С покрытием кода
```bash
pytest --cov=.
```

### С подробным выводом
```bash
pytest -v --tb=long
```

## Моки и фикстуры

### Используемые моки

- **unittest.mock.patch**: Для замены зависимостей
- **unittest.mock.AsyncMock**: Для асинхронных функций
- **unittest.mock.Mock**: Для объектов и методов

### Фикстуры

- **setup_method/teardown_method**: Создание/удаление временных файлов
- **tempfile.NamedTemporaryFile**: Временные конфигурационные файлы

## Лучшие практики тестирования

### Принципы
- **Изоляция**: Каждый тест независим
- **Повторяемость**: Тесты дают одинаковый результат
- **Читабельность**: Названия тестов описывают поведение
- **Полнота**: Покрытие основных сценариев использования

### Паттерны
- **Arrange-Act-Assert**: Подготовка - Действие - Проверка
- **Given-When-Then**: Дано - Когда - Тогда
- **Happy Path + Edge Cases**: Основные сценарии + граничные случаи

## Добавление новых тестов

### Структура нового теста

```python
def test_new_functionality(self):
    """Описание того, что тестируется"""
    # Arrange - подготовка данных
    # Act - выполнение действия
    # Assert - проверка результата
```

### Добавление моков

```python
@patch('module.Class')
def test_with_mock(self, mock_class):
    # Настройка мока
    mock_instance = Mock()
    mock_class.return_value = mock_instance
```

## Устранение неисправностей

### Распространенные проблемы

#### ImportError
**Причина:** Неправильные пути импорта
**Решение:** Проверить PYTHONPATH и структуру проекта

#### AsyncioError
**Причина:** Неправильная настройка asyncio в тестах
**Решение:** Добавить декоратор `@pytest.mark.asyncio`

#### FileNotFoundError
**Причина:** Отсутствующие тестовые файлы
**Решение:** Проверить создание временных файлов в setup_method

### Полезные команды

```bash
# Показать все доступные тесты
pytest --collect-only

# Запустить с отладкой
pytest -s --pdb

# Запустить только упавшие тесты
pytest --lf

# Показать самый медленный тест
pytest --durations=10
```
