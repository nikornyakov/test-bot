# 🤖 Test Bot

Telegram бот для тестирования и демонстрации архитектуры. Проект является полной копией lets-play-basketbot с идентичной структурой и функциональностью.

## 📋 Описание

Этот проект представляет собой тестовый Telegram бот, который демонстрирует современную архитектуру и лучшие практики разработки. Проект является полной копией lets-play-basketball с идентичной структурой и функциональностью для целей тестирования и обучения.

Бот автоматически:

- Отправляет опросы по понедельникам и средам о посещении предстоящих тренировок
- Рассылает напоминания по вторникам и четвергам
- Отправляет приветственные сообщения с актуальным расписанием
- Ведет подробное логирование всех действий

## ✨ Возможности

### 🤖 Автоматические сообщения

- **Напоминания**: По вторникам и четвергам бот отправляет напоминания о предстоящих тренировках
- **Приветственные сообщения**: Современные приветственные сообщения с подробной информацией о команде

## 📦 Установка

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/nikornyakov/lets-play-basketball.git
   cd lets-play-basketball
   ```

2. **Создайте виртуальное окружение:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # Для Linux/Mac
   # или
   venv\Scripts\activate     # Для Windows
   ```

3. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Настройте конфигурацию:**
   ```bash
   cp .env.example .env
   # Отредактируйте .env с вашими данными
   ```

5. **Проверьте тесты:**
   ```bash
   pytest tests/ -v
   ```

## ⚙️ Настройка

### Переменные окружения

Создайте файл `.env` в корне проекта:

```bash
# Токен вашего Telegram бота (получить у @BotFather)
BOT_TOKEN=your_bot_token_here

# ID группы, куда будут отправляться сообщения
GROUP_ID=-1001234567890
```

### Конфигурация

Проект использует JSON-конфигурацию для настройки сообщений, расписания и других параметров. Основной файл конфигурации `config.json` содержит:

- **schedule**: Расписание тренировок и опросов
- **venue**: Информация о месте проведения
- **messages**: Шаблоны всех сообщений бота
- **team**: Информация о команде
- **logging**: Настройки логирования

### Как получить BOT_TOKEN

1. Напишите [@BotFather](https://t.me/botfather) в Telegram
2. Используйте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Скопируйте токен из ответа BotFather

### Как получить GROUP_ID

Запустите скрипт для определения ID группы:

```bash
python get_group_id.py
```

Или используйте GitHub Action "Get Group ID 2" в разделе Actions репозитория.

## 🏗 Архитектура

Проект построен по современной модульной архитектуре с полным разделением ответственности:

### 📦 Основные модули

- **`bot_base.py`**: Базовый класс `TelegramBotBase` с общей функциональностью
  - Инициализация бота и работа с токенами
  - Отправка сообщений и опросов
  - Обработка ошибок и логирование

- **`bot_config.py`**: Класс `Config` для работы с JSON-конфигурацией
  - Загрузка и валидация конфигурации
  - Удобные методы доступа к настройкам
  - Дефолтные значения и обработка ошибок

- **`simple_bot.py`**: Основная логика бота
  - Функции отправки приветственных сообщений
  - Опросы посещаемости по расписанию
  - Напоминания о тренировках

- **`get_group_id.py`**: Утилита для получения ID групп
  - Поиск групп, где добавлен бот
  - Отображение информации о чатах

- **`test_poll.py`**: Тестовый скрипт для проверки работоспособности

### ⚙️ Конфигурационная система

- **`config.json`**: Основной файл конфигурации
  - Расписание тренировок и опросов
  - Шаблоны всех сообщений
  - Информация о команде и месте проведения
  - Настройки логирования

- **`.env`**: Переменные окружения для чувствительных данных
  - `BOT_TOKEN`: Токен Telegram бота
  - `GROUP_ID`: ID группы для отправки сообщений

### 🧪 Система тестирования

- **`tests/`**: Директория с тестами
  - `test_bot_config.py`: Тесты конфигурации
  - `test_bot_base.py`: Тесты базового класса
  - `test_simple_bot.py`: Интеграционные тесты

- **`pytest.ini`**: Конфигурация pytest с поддержкой asyncio

### 🎯 Особенности архитектуры

- **Наследование**: Все компоненты наследуются от базовых классов
- **Конфигурируемость**: Все тексты и настройки вынесены в конфигурацию
- **Тестируемость**: Полная система тестирования с моками и фикстурами
- **Расширяемость**: Легкое добавление новых функций через конфигурацию
- **Надежность**: Graceful обработка ошибок и валидация данных

## 🧪 Тестирование

Проект имеет полную систему тестирования с использованием pytest:

### 🏃 Запуск тестов

```bash
# Все тесты
pytest tests/ -v

# Конкретный файл
pytest tests/test_simple_bot.py -v

# С покрытием кода
pytest --cov=. tests/

# Только упавшие тесты
pytest --lf
```

### 📊 Типы тестов

- **Unit тесты**: Тесты отдельных классов и функций

### 🎯 Покрытие тестами

- **bot_base.py**: Тесты TelegramBotBase и утилитарных функций

### 🔧 Конфигурация тестов

- **pytest.ini**: Настройки pytest с поддержкой asyncio

### 🧪 Написание тестов

```python
# Пример теста
import pytest
from unittest.mock import AsyncMock, patch

class TestNewFeature:
    @pytest.mark.asyncio
    async def test_new_function(self):
        # Arrange
        mock_bot = AsyncMock()
        
        # Act
        result = await new_function(mock_bot)
        
        # Assert
        assert result is True
        mock_bot.send_message.assert_called_once()
```
