# 🚀 Настройка GitHub Actions для автоматических отчетов

## 📋 Что это?

Автоматическая система сбора данных из Telegram и генерации отчетов через GitHub Actions.

## 🎯 Возможности

### 📅 Автоматические запуски:
- **Ежедневно** в 20:00 МСК - сбор данных
- **Еженедельно** (воскресенье 20:00 МСК) - недельный отчет
- **Ежемесячно** (1 числа 10:00 МСК) - месячный отчет
- **Ежегодно** (1 января 12:00 МСК) - годовой отчет

### 🎮 Ручной запуск:
- Сбор данных
- Недельный/месячный/годовой отчет
- Тестирование системы

## 🚀 Запуск

### Автоматический запуск

Workflow запускается автоматически по расписанию:
- Каждый день в 20:00 МСК
- Каждое воскресенье в 20:00 МСК
- 1 числа каждого месяца в 10:00 МСК
- 1 января в 12:00 МСК

### Ручной запуск

1. Перейдите в **Actions** → **Simple Poll Collection and Reports**
2. Нажмите **Run workflow**
3. Выберите параметры:
   - **Действие**: что сделать
   - **Год**: для годовых отчетов

#### Варианты действий:
- `collect` - только сбор данных
- `report_weekly` - недельный отчет
- `report_monthly` - месячный отчет
- `report_yearly` - годовой отчет
- `report_test` - тестирование системы

## 📁 Результаты

### Артефакты
Каждый запуск создает артефакт `polls-data-{номер}` с:
- `reports/polls.json` - данные опросов
- `reports/users.json` - пользователи
- Логи выполнения

### Коммиты
Данные автоматически коммитятся в репозиторий:
```

```

## 🚨 Troubleshooting

### "Bot not found"
**Причина**: Неверный BOT_TOKEN
**Решение**: Обновите secret `BOT_TOKEN`

### "Chat not found"
**Причина**: Неверный GROUP_ID
**Решение**: 
1. Получите реальный ID группы
2. Обновите secret `GROUP_ID`

### "No polls found"
**Причина**: Бот не видит опросы
**Решение**:
1. Убедитесь что бот в группе
2. Отправьте /start в группе
3. Создайте новый опрос

### "Permission denied"
**Причина**: Нет прав на push
**Решение**: Проверьте `permissions: contents: write` в workflow

  uses: 8398a7/action-slack@v3
